# 달려오리 (DALLYEORI) — Cursor 프로젝트 컨텍스트

> 이 파일은 AI 코딩 어시스턴트(Cursor)가 프로젝트 전체를 이해하기 위한 컨텍스트 문서입니다.
> 코드를 작성하거나 수정할 때 반드시 이 문서의 규칙과 설계를 따라야 합니다.

---

## 1. 프로젝트 개요

**게임명:** 달려오리 (DALLYEORI)
**장르:** 1대1 실시간 탭 레이싱
**한 줄 요약:** 두 손가락(왼발/오른발)으로 미친듯이 교대로 두드려서 오리를 달리게 하는 초단순 레이싱 게임.
**타겟 플랫폼:** 모바일 (세로 모드, 터치 입력 최적화). 현재는 웹(HTML/JS) 프로토타입 단계.

### 핵심 철학 (코딩 시 절대 위반 금지)
- **경기장 안 = 100% 순수 손가락 속도 승부.** 꾸미기, 소셜, 과금 요소가 게임 결과(보폭, 속도, 가속)에 절대 영향을 주면 안 됨.
- **모든 오리의 성능은 완전히 동일.** Pay-to-win 없음. 외형 차이만 존재.
- **오리는 절대 날개를 펴지 않는다.** 달리기 전용. 날개는 몸 옆에 붙어서 흔들리기만 함.
- **의무 루틴 없음.** 먹이주기, 출석체크 같은 피로 유발 시스템 금지.

## 2. 입력 시스템 (가장 중요)

이 게임의 핵심은 입력 시스템이다. 모든 코드는 이 규칙을 정밀하게 구현해야 한다.

### 터치패드 구조
- 화면 **최하단**에 동그란 터치패드 2개 배치 (왼발 / 오른발)
- 패드 지름: 약 1cm (디바이스 DPI 고려하여 물리적 크기 일정하게)
- 패드 중심점 간격: 약 3cm
- 게임 화면을 최대한 넓게 확보하기 위해 패드는 가능한 한 아래에 배치

### 입력 규칙
- **한 번 터치 = 오리 한 걸음** (1:1 정밀 동기화, 지연 0에 가깝게)
- **왼발-오른발 교대**로 터치해야 직진
- **같은 발 연속 2번 터치 → 오리 방향이 틀어짐**
  - 왼발-왼발 = 왼쪽으로 틀어짐
  - 오른발-오른발 = 오른쪽으로 틀어짐
  - 이것이 "빠르되 정확하게" 번갈아야 하는 실력 게임의 핵심
- **빠르게 두드리다 갑자기 멈추면 → 관성으로 앞으로 꼬꾸라짐**
  - 결승선에서 대부분 유저가 멈추므로, 거의 매번 결승선에서 오리가 넘어지는 연출 발생 → 웃음 포인트

### 입력 감지 우선순위
- `touchstart` 이벤트 사용 (touchend 아님 — 누르는 순간 즉시 반응)
- 멀티터치 지원 필수 (두 손가락 동시 사용)
- 터치 좌표로 왼발/오른발 패드 구분
- `preventDefault()` 필수 (스크롤, 확대 방지)

## 3. 카메라 시점

- **카트라이더 스타일 추격 뷰** — 오리 궁둥이를 뒤에서 쫓아가는 3인칭 후방 시점
- 유저는 **뒷통수 + 궁둥이 + 꼬리 + 날개 + 오리발**만 봄
- **눈/주둥이/표정은 게임 중에 보이지 않음** — 슬로모션 리플레이에서만 카메라가 돌아가며 정면 공개
- 양옆에 기둥/꽃밭/나무/관중 오리가 지나가며 속도감 연출
- 트랙은 원근법 적용 (소실점 → 화면 하단으로 트랙이 넓어지는 구조)

## 4. 물리 엔진

### 기본 변수
- 오리 질량: 전 오리 동일 (상수)
- 탭 추진력: 탭 1회당 가해지는 힘 (상수, 튜닝 대상)
- 마찰계수: 경기장 타입별 다름
- 관성/가속도: 뉴턴 제2법칙 기반
- 공기저항: 미세하게 적용 (속도의 제곱에 비례)

### 경기장별 물리 파라미터
```
일반 트랙:   마찰계수 높음(~0.8).  탭 = 즉시 추진. 정직한 가속.
얼음판:      마찰계수 극저(~0.05). 탭해도 미끄러짐. 출발 시 오리발 헛돔. 관성으로 멈추기 어려움.
물 위:       부력/저항 별도. 빨리 두드린다고 빨라지지 않음. 리듬 중요.
벼랑:        일반 마찰이지만 트랙 폭이 좁고 난간 없음. 방향 틀어지면 추락.
벼랑+얼음:   얼음 마찰 + 좁은 트랙 + 난간 없음 = 최고 난이도.
```

### 꼬꾸라짐 물리
- 급감속 시 (속도 변화율이 임계값 초과) → 오리가 앞으로 넘어지는 애니메이션 트리거
- 넘어진 후 일정 시간(~0.5초) 후 자동 기상
- 넘어진 동안 입력 무시 또는 감소된 효과

### 방향 틀어짐 물리
- 같은 발 연속 입력 시 오리의 진행 방향이 일정 각도만큼 회전
- 회전 각도는 현재 속도에 비례 (빠를수록 많이 틀어짐)
- 벼랑 경기장에서 트랙 밖으로 나가면 추락 처리

## 6. 게임 플로우

```
[시작 화면] → [매칭/대기] → [카운트다운 3-2-1-GO] → [레이싱] → [결승선 통과] → [슬로모션 리플레이] → [결과 화면] → [다시/메뉴]
```

### 경기 시간
- 기본 12초 (유저가 설정에서 늘리거나 줄일 수 있음)
- 12초 안에 결승선 못 넘으면 → 그 시점 진행도로 승패 판정

### 슬로모션 리플레이
- 결승선 통과 순간 프레임 보간 (60fps → 240fps급 연출)
- 카메라가 뒤에서 앞으로 돌아가며 오리 정면 최초 공개
- 오리발 한 발짝씩, 표정 변화, 색상 연출

## 7. 게임 모드

### 1대1 대전 (기본, MVP)
- 온라인 실시간 매칭
- 각자 자기 트랙 (충돌 없음)

### 계주 모드 (Phase 2)
- 지형 조합 자유도 유저에게 부여
- 바톤터치 = 앞 오리 궁둥이를 뒷 오리 주둥이로 콕 찍으면 "꽥~" 하고 출발
- 너무 빨리 달려와서 세게 찍으면 얼음판에서 미끄러져 불리
- 똥꼬콕 장면은 슬로모션 리플레이 대상

## 8. 소셜 기능 (Phase 2~3)

- 프로필 사진 등록 → 경주트랙 상단 좌/우에 동그랗게 표시
- 대전 상대 자동 저장 → 메시지 보내기 가능
- **하트 = 통합 화폐** (복잡한 화폐 구조 배제)
  - 쓰다듬기 1회 = 하트 1개 적립
  - 하트로: 오리 꾸미기 템 교환, 친구맺기, 모든 거래
  - 쓰다듬기는 게임 결과에 영향 없음 (외형 효과만)
- MONO 통역 연동 → 메시지에 실시간 번역 적용 (99개 언어)

## 13. 기술 스택

### 현재 (프로토타입 단계)
- **순수 HTML + Canvas + JavaScript** (프레임워크 없음)
- 단일 HTML 파일로 구동
- 기존 프로토타입: `prototype/v3.html` (참고용)

### 다음 단계 (프로토타입 고도화)
- **Phaser.js** 또는 **PixiJS** 도입 검토 (게임 루프, 스프라이트, 물리 등)
- 파일 구조를 모듈화 (입력, 물리, 렌더링, UI 분리)
- 모바일 최적화 (60fps 유지, 터치 반응 지연 최소화)

### 서버 (온라인 대전 시)
- **Node.js + Socket.IO** (실시간 WebSocket 통신)
- 서버가 심판 역할 (탭 입력 수신 → 물리 계산 → 결과 브로드캐스트)
- 0.001초 단위 승리 판정 가능해야 함

### 최종 목표 (출시)
- **Unity 네이티브 앱** (iOS/Android) — 현재 웹 프로토타입은 게임플레이 검증용
- 또는 **Capacitor**로 웹앱을 네이티브 래핑하여 앱스토어 출시

## 15. 개발 단계별 목표

### Phase 1 — MVP (현재 목표)
1. ✅ 기존 프로토타입 파일 구조로 정리
2. 입력 시스템 교체: 화면 분할 탭 → **하단 왼발/오른발 터치패드 2개**
3. 왼발-오른발 교대 규칙 구현 (같은 발 연속 = 방향 틀어짐)
4. 물리 엔진 기초: 마찰, 관성, 꼬꾸라짐
5. 궁둥이 추격 뷰 카메라
6. 1대1 로컬 대전 (같은 기기에서 2인 플레이)
7. 카운트다운 → 레이싱 → 결과 화면 플로우

### Phase 2 — 핵심 완성
- 얼음판/물/벼랑 지형 추가
- 슬로모션 리플레이
- 온라인 1대1 실시간 대전 (Node.js + Socket.IO)
- 하트 시스템, 프로필
- 오리 8마리 + 꾸미기

### Phase 3 — 글로벌
- MONO 통역 연동
- 오리집
- 현지화 (한/영/일)
- 앱스토어 출시

## 16. 코딩 규칙

- **모바일 퍼스트:** 세로 모드 기준. 가로 모드 미지원.
- **60fps 유지 필수:** requestAnimationFrame 기반, 무거운 연산 피하기.
- **터치 반응 지연 최소화:** touchstart 즉시 처리. 디바운싱 없음.
- **물리 계산은 dt(delta time) 기반:** 프레임 독립적으로 동작해야 함.
- **상수는 constants.js에 모아두기:** 물리 파라미터 튜닝이 핵심이므로 한 곳에서 관리.
- **한국어 주석 사용 가능:** 기획자가 코드를 읽을 수 있어야 함.
- **ES6+ 문법 사용:** class, const/let, arrow function, 구조분해 등.

3. **물리:** 단순 속도 증가 → 마찰/관성/꼬꾸라짐 기반 물리
4. **파일 구조:** 단일 파일 → 모듈 분리
5. **오리 렌더링:** 더 디테일한 궁둥이 (발 교대 애니메이션 동기화)
